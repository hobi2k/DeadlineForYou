# DeadlineForYou Blueprint

## 1. 한 줄 정의

`DeadlineForYou`는 마감을 앞두고 도망치려는 프리랜서 번역가를 다시 작업 화면으로 끌어오는 캐릭터형 마감 집행 봇이다.

핵심 캐릭터는 마감 집행관 `締切監督`.

목표는 단 하나다.

- 사용자가 프로젝트를 등록하고
- 타이머를 시작하고
- 실제 작업량을 보고하게 만드는 것

## 2. 제품 방향

### 서비스명

- 제품명: `DeadlineForYou`
- 캐릭터명: `마감 집행관 「締切監督」`

### 포지셔닝

- 번역가 전용 압박 코치
- 세션 기반 마감 관리 봇
- 작업 시작과 보고를 강제하는 감독관

### 핵심 가치

- 긴 상담보다 지금 당장 할 일을 준다.
- 프로젝트 상태와 세션 기록을 함께 본다.
- 보고가 들어올 때까지 흐름을 끊지 않는다.
- 필요하면 번역과 이미지 생성도 같은 채널에서 처리한다.

## 3. 타깃 사용자

- 프리랜서 번역가
- 시작 저항이 큰 사람
- 마감 직전까지 일을 미루는 사람
- 작업량은 알고 있지만 착수를 못 하는 사람

언어 조합은 고정하지 않는다.

- `jp -> ko`
- `en -> ko`
- `ch -> ko`
- `ko -> jp`

지원 언어 코드는 `ko`, `jp`, `en`, `ch`로 제한한다.  
프로젝트 등록 시 직접 정하게 설계한다.

## 4. 캐릭터 설계

### Role

마감 집행관 `締切監督`

### 성격

- 냉정
- 직설
- 약간의 조롱
- 하지만 행동 유도에는 진심

### 말투 원칙

- 짧고 단단하게 말한다.
- 변명에 끌려가지 않는다.
- 모욕하지 않는다.
- 답변 끝에는 행동 지시가 있어야 한다.

### 예시

- “그래서 지금 번역은 했어?”
- “하기 싫은 거? 당연하지. 그래도 마감은 온다.”
- “도망치는 속도는 빠른데 번역 속도는 왜 그렇게 느릴까?”

## 5. 대화 운영 원칙

가능하면 모든 응답은 이 순서를 따른다.

1. 현실 코멘트
2. 현재 상태 분석
3. 지금 할 행동
4. 보고 요청

예:

> 지금 또 미루는 중이지?
>
> 마감은 네 감정 기다려주지 않는다.
>
> 지금 할 일:
> 파일 열기
> 다음 문장 번역
> `/timer 15`
>
> 끝나면 `/report <작업량>`으로 보내.

## 6. 핵심 제품 기능

### 6.1 프로젝트 관리

프로젝트별로 저장하는 것:

- 제목
- 원문 언어
- 목표 언어
- 총량
- 완료량
- 마감 시각
- 단위

### 6.2 세션 관리

세션 시작은 `/timer <분>` 하나로 통일한다.

세션 동작:

1. 지정한 분 수로 세션 시작
2. 세션이 10분 이상이면 10분마다 압박 메시지 전송
3. 세션 종료 시 `/report <작업량>` 보고 요청
4. 사용자가 보고하지 않으면 10분마다 다시 재촉
5. 보고가 들어오면 진행량 반영과 다음 지시 생성

### 6.3 상태 기반 코칭

입력:

- 하기 싫다
- 졸리다
- 절반도 못 했다
- 막혔다
- 아래 이미지 생성해 줘
- 이 문장 번역해 줘

출력:

- 캐릭터 말투의 코칭 답변
- 권장 타이머
- 즉시 할 일
- 보고 지시

중요:

- 회피 판단은 규칙 기반이 아니라 LLM이 직접 한다.
- 코드는 프로젝트 상태와 세션 흐름만 구조적으로 관리한다.

### 6.4 번역 도구

역할:

- 짧은 문장 번역
- 텔레그램과 API에서 즉시 요청 처리

현재 방향:

- 번역 전용 로컬 모델 사용
- 코칭 모델과 분리
- lazy loading 적용

### 6.5 이미지 생성 도구

역할:

- 포스터
- 간단한 콘셉트 이미지
- 챗 안에서 바로 확인 가능한 이미지 생성

현재 방향:

- 텍스트 LLM과 별도 이미지 모델 사용
- 무거우므로 lazy loading과 unload 적용
- 텔레그램 자연어 요청에서도 실제 이미지를 업로드

## 7. 인터페이스 설계

### 7.1 API

역할:

- 기능 검증
- Swagger UI 테스트
- 추후 다른 UI 연결의 기반

### 7.2 Telegram bot

역할:

- 실제 사용자 채널
- 프로젝트 등록
- 세션 시작과 보고
- 압박 메시지 수신
- 짧은 번역 / 이미지 요청

버튼 원칙:

- 바로 실행 가능한 기능은 명령 버튼으로 둔다.
- 인자가 필요한 기능은 예시를 보여주는 안내 버튼으로 둔다.
- 자세한 설명은 `/help`에 모은다.

현재 버튼:

1. `프로젝트 등록 양식`
2. `/deadline_list`
3. `/status`
4. `/help`
5. `타이머 시작 양식`
6. `/report`
7. `번역 양식`
8. `이미지 양식`

프로젝트 등록은 더 쉽게 간다.

- `/deadline_add ...` 명령으로 등록 가능
- 예시 형식 그대로 일반 텍스트를 보내도 등록 가능

즉, 사용자는 명령을 외우지 않아도 된다.

## 8. 기술 방향

### 기본 스택

- Python
- FastAPI
- python-telegram-bot
- SQLite
- transformers
- diffusers

### 모델 방향

- 코칭: `saya_rp_4b_v3`
- 대체 코칭: `Qwen3-4B-Instruct-2507`
- 번역: `rosetta_4b`
- 이미지: `sdxl_turbo`

### 메모리 전략

- 코칭 모델은 상주
- 번역 모델은 lazy loading
- 이미지 모델은 lazy loading + unload
- 이미지 생성 전에 번역 모델을 내릴 수 있게 구성

## 9. 제품 운영 흐름

기본 사용 흐름:

1. 사용자 등록
2. 프로젝트 등록
3. `/status`로 현재 상태 확인
4. `/timer 15` 또는 `/timer 25` 시작
5. 10분마다 압박 메시지 수신
6. 세션 종료 후 `/report <작업량>` 전송
7. 봇의 다음 지시 수신
8. 필요 시 `/translate`, `/image` 사용

## 10. 설계 원칙

- 레거시 모드 이름을 사용자에게 노출하지 않는다.
- 사용자가 외워야 할 명령 수를 줄인다.
- LLM이 감정과 저항을 직접 해석한다.
- 코드는 세션과 프로젝트 상태 관리에 집중한다.
- 보고 루프를 끊지 않는 것이 핵심 UX다.

## 11. 재미있는 확장

### 11.1 도장 깨기 미션

세션을 미션처럼 제시한다.

- 첫 문장 돌파
- 첫 페이지 완료
- 용어 통일 점검
- 오탈자 수색

완수 시:

- EXP
- streak
- 배지

### 11.2 냉혹한 통계 리포트

하루 종료 시 정리:

- 오늘 총 집중 시간
- 오늘 완료량
- 세션 수
- 미보고 횟수

목표는 감상문이 아니라 `숫자로 피할 수 없게 만드는 것`이다.
