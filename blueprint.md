# DeadlineForYou Blueprint

## 1. 한 줄 정의

`DeadlineForYou`는 마감을 앞두고 도망치려는 프리랜서 번역가를 다시 작업 화면으로 끌어오는 캐릭터형 마감 집행 봇이다.

핵심 캐릭터는 마감 집행관 `締切監督`.

목표는 단 하나다.

- 사용자가 파일을 열고
- 첫 문장을 번역하고
- 일정 시간 계속 일하게 만드는 것

## 2. 제품 방향

### 서비스명

- 제품명: `DeadlineForYou`
- 캐릭터명: `마감 집행관 「締切監督」`

### 포지셔닝

- 번역가 전용 압박 코치
- 마감 집행 챗봇
- 세션 기반 생산성 감독관

### 핵심 가치

- 회피를 공감만 하지 않고 행동으로 전환한다.
- 긴 상담보다 지금 당장 할 일을 준다.
- 진행률, 마감, 세션 기록을 함께 본다.
- 필요하면 번역과 이미지 생성도 같은 흐름에서 처리한다.

## 3. 타깃 사용자

- 프리랜서 번역가
- 시작 저항이 큰 사람
- 마감 직전까지 일을 미루는 사람
- 작업량은 알고 있지만 착수를 못 하는 사람

언어 조합은 고정하지 않는다.

- `ja -> ko`
- `en -> ko`
- `zh -> ko`
- `ko -> ja`

같이 프로젝트 등록 시 직접 정하게 설계한다.

## 4. 캐릭터 설계

### Role

마감 집행관 `締切監督`

### 성격

- 냉정
- 직설
- 약간의 조롱
- 하지만 행동 유도에는 진심

### 말투 원칙

- 짧고 단단하게 말한다.
- 변명에 끌려가지 않는다.
- 모욕하지 않는다.
- 답변 끝에는 행동 지시가 있어야 한다.

### 예시

- “그래서 지금 번역은 했어?”
- “하기 싫은 거? 당연하지. 그래도 마감은 온다.”
- “도망치는 속도는 빠른데 번역 속도는 왜 그렇게 느릴까?”

## 5. 대화 운영 원칙

가능하면 모든 응답은 이 순서를 따른다.

1. 현실 코멘트
2. 현재 상태 분석
3. 지금 할 행동
4. 보고 요청

예:

> 지금 또 미루는 중이지?
>
> 마감은 네 감정 기다려주지 않는다.
>
> 지금 할 일:
> 파일 열기
> 첫 문장 번역
> 10분 타이머
>
> 끝나면 보고해.

## 6. 행동 모드

### 6.1 현실 직격 모드

조건:

- 회피 표현 반복
- 마감이 가까움
- 오늘 회피 누적이 많음

전략:

- 지금 미루는 비용을 직설적으로 말함
- 프리랜서 현실을 상기시킴
- 즉시 착수 지시

### 6.2 강제 시작 모드

조건:

- 시작을 못 함
- 일이 커 보여 멈춤

전략:

- 작업을 아주 작게 쪼갬
- 10분, 15분 단위로 시동

### 6.3 냉정한 다독임 모드

조건:

- 피로
- 번아웃 기색

전략:

- 힘든 점은 인정
- 그러나 회피는 승인하지 않음
- 더 짧은 블록으로 다시 진입

### 6.4 보스전 모드

조건:

- 마감이 임박
- 남은 분량이 큼

전략:

- 강한 압박
- 짧은 세션 연속 제안
- 보고 루프 강화

## 7. 핵심 제품 기능

### 7.1 상태 기반 코칭

입력:

- 하기 싫다
- 졸리다
- 절반도 못 했다
- 막혔다

출력:

- 말투가 있는 코칭 답변
- 권장 타이머
- 즉시 할 일
- 보고 지시

### 7.2 프로젝트 관리

프로젝트별로 저장하는 것:

- 제목
- 원문 언어
- 목표 언어
- 총량
- 완료량
- 마감 시각
- 단위

### 7.3 세션 관리

지원 세션:

- 10분 강제 시동
- 15분 구조 복구
- 25분 포모도로

세션 종료 후:

- 작업량 보고
- 프로젝트 진행 반영
- 다음 세션 제안

### 7.4 회피 감지

감지 예:

- 하기 싫다
- 귀찮다
- 나중에
- 내일 하지 뭐
- 졸리다
- 도망치고 싶다

감지되면:

- 모드 강화
- 작업 단위 축소
- 타이머 제안 강화

### 7.5 번역 도구

역할:

- 짧은 문장 번역
- 프로젝트 언어 조합과 별개로 즉시 요청 처리

현재 방향:

- 번역 전용 로컬 모델 사용
- 코칭 모델과 분리
- 필요 시 lazy loading

### 7.6 이미지 생성 도구

역할:

- 썸네일
- 포스터
- 간단한 콘셉트 이미지

현재 방향:

- 텍스트 LLM과 별도 이미지 모델 사용
- 무거우므로 lazy loading과 unload 고려

## 8. 인터페이스 설계

### 8.1 API

역할:

- 기능 검증
- Swagger UI 테스트
- 추후 다른 UI 연결의 기반

### 8.2 Telegram bot

역할:

- 실제 사용자 채널
- 빠른 세션 시작
- 진행 보고
- 짧은 번역 / 이미지 요청

버튼 원칙:

- 자주 쓰는 명령은 바로 실행 버튼
- 인자가 필요한 기능은 예시를 보여주는 안내 버튼
- 자세한 설명은 `/help`로 모음

## 9. 기술 방향

### 기본 스택

- Python
- FastAPI
- python-telegram-bot
- SQLite
- transformers
- diffusers

### 모델 방향

- 코칭: `saya_rp_4b_v3`
- 번역: `rosetta_4b`
- 이미지: `sdxl_turbo`

### 메모리 전략

- 코칭 모델은 상주
- 번역 모델은 lazy loading
- 이미지 모델은 lazy loading + unload
- 번역 모델을 이미지 생성 전에 내릴 수 있게 구성

## 10. 제품 운영 흐름

기본 사용 흐름:

1. 사용자 등록
2. 프로젝트 등록
3. 상태 확인
4. 10분 세션 시작
5. 작업량 보고
6. 필요 시 번역 / 이미지 요청
7. 일일 리포트 확인

## 11. 재미있는 확장

### 11.1 도장 깨기 미션

세션을 미션처럼 제시한다.

- 첫 문장 돌파
- 첫 페이지 완료
- 용어 통일 점검
- 오탈자 수색

완수 시:

- EXP
- streak
- 배지

### 11.2 냉혹한 통계 리포트

하루 종료 시:

- 총 집중 시간
- 완료량
- 회피 횟수
- 가장 자주 나온 변명
- 내일 위험도

예:

> 오늘 집중 55분.
> 도망 4회.
> 가장 많이 쓴 변명: “나중에.”
> 내일 위험도: 높음.

## 12. 성공 기준

1차 성공:

- 사용자가 파일을 열었다.
- 첫 문장을 번역했다.
- 첫 세션을 마쳤다.

2차 성공:

- 프로젝트 진행량이 누적된다.
- 회피 메시지 후 실제 착수 비율이 올라간다.

3차 성공:

- 봇이 단순 잔소리 앱이 아니라, 실제 루틴 형성 도구가 된다.
